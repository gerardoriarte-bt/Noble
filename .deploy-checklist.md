# ✅ Checklist de Despliegue - Noble.com

## 📋 Antes de Desplegar

### 1. Preparación Local
- [x] Código actualizado y probado localmente
- [x] Build de producción funciona (`npm run build`)
- [x] Todos los archivos de configuración creados
- [x] Scripts tienen permisos de ejecución

### 2. AWS EC2
- [ ] Instancia EC2 creada (Ubuntu 22.04 LTS)
- [ ] Security Group configurado:
  - [ ] Puerto 80 (HTTP) abierto
  - [ ] Puerto 443 (HTTPS) abierto
  - [ ] Puerto 22 (SSH) abierto (preferiblemente solo tu IP)
- [ ] Key Pair descargado y guardado de forma segura
- [ ] IP pública de la instancia anotada

### 3. DNS
- [ ] Dominio noble.com configurado
- [ ] Registro A para `noble.com` → IP de EC2
- [ ] Registro A para `www.noble.com` → IP de EC2
- [ ] DNS propagado (verificar con `dig noble.com`)

## 🚀 Proceso de Despliegue

### Paso 1: Conectar a EC2
```bash
ssh -i "tu-key.pem" ubuntu@tu-ip-publica
```

### Paso 2: Subir Archivos
```bash
# Desde tu máquina local
scp -i "tu-key.pem" ec2-setup-noble.sh ubuntu@tu-ip-publica:~/
scp -i "tu-key.pem" nginx-noble.conf ubuntu@tu-ip-publica:~/
scp -i "tu-key.pem" deploy-noble.sh ubuntu@tu-ip-publica:~/
scp -i "tu-key.pem" setup-domain-noble.sh ubuntu@tu-ip-publica:~/
```

### Paso 3: Configuración Inicial
```bash
# En el servidor
chmod +x *.sh
sudo bash ec2-setup-noble.sh
```

### Paso 4: Configurar Dominio y SSL
```bash
# En el servidor (después de que DNS esté propagado)
sudo bash setup-domain-noble.sh
```

### Paso 5: Desplegar Aplicación
```bash
# En el servidor
git clone git@github.com:gerardoriarte-bt/Noble.git
cd Noble
bash deploy-noble.sh
```
